# Security Policy

## How to Report Vulnerabilities
Please report vulnerabilities privately via GitHub Security Advisory (preferred) or direct maintainer contact.

Do not disclose details publicly before a fix or mitigation is prepared.

## What Should Be Reported
Please report issues involving:
- authentication and authorization logic
- token generation/validation and session security
- transfer and balance integrity
- rate limiting and abuse protections
- dependency and supply-chain vulnerabilities

## Scope
This policy applies to this repository, including API, use-case, domain, infrastructure, and worker modules.

## Response Expectations
- Initial triage target: within 5 business days.
- Confirmed issues will receive a remediation plan and coordination timeline.
- Coordinated disclosure is preferred once a fix is available.

## Project Context
Backend-Path is an actively improved portfolio/open-source project. Security hardening is ongoing and responsible disclosure is appreciated.
